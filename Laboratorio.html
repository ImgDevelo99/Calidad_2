Introducción a los Estándares de Calidad de Software

Este laboratorio interactivo te permitirá explorar los estándares clave utilizados para asegurar y evaluar la calidad del software. Nos enfocaremos en ISO/IEC 25000 (SQuaRE), ISO/IEC 15504 (SPICE) y estándares relevantes de IEEE, a un nivel intermedio.

La calidad del software no es solo que funcione, sino que cumpla con una serie de atributos (funcionalidad, fiabilidad, usabilidad, etc.) y que los procesos utilizados para crearlo sean adecuados y predecibles. Estos estándares proporcionan marcos y modelos para lograrlo.

ISO/IEC 25000 (SQuaRE)

ISO/IEC 25000 (SQuaRE - System and Software Quality Requirements and Evaluation) es una familia de normas que establece un modelo de calidad del producto software, definiendo características y subcaracterísticas de calidad, y proporciona guías para la evaluación.

Las principales características de calidad del modelo son:

  • Idoneidad Funcional: Completitud, Correctitud, Pertinencia.
  • Eficiencia de Desempeño: Comportamiento temporal, Utilización de recursos, Capacidad.
  • Compatibilidad: Coexistencia, Interoperabilidad.
  • Usabilidad: Reconocimiento de la idoneidad, Aprendizaje, Operabilidad, Protección frente a errores de usuario, Estética, Accesibilidad.
  • Fiabilidad: Madurez, Disponibilidad, Tolerancia a fallos, Capacidad de recuperación.
  • Seguridad: Confidencialidad, Integridad, No repudio, Responsabilidad, Autenticidad.
  • Mantenibilidad: Modularidad, Reusabilidad, Analizabilidad, Modificabilidad, Capacidad para ser probado.
  • Portabilidad: Adaptabilidad, Instalabilidad, Coexistencia, Capacidad de reemplazo.

ISO/IEC 15504 (SPICE)

ISO/IEC 15504 (ahora parte de la serie ISO 330xx, pero comúnmente conocida como SPICE - Software Process Improvement and Capability Determination) es un marco para la evaluación de procesos de desarrollo de software. No evalúa el *producto* software directamente, sino los *procesos* utilizados para crearlo.

SPICE define un modelo de referencia de procesos (PRM) y un modelo de evaluación de procesos (PAM). La evaluación determina el nivel de capacidad de un proceso utilizando una escala de 6 niveles:

  • Nivel 0: Incompleto: El proceso no se realiza o no alcanza su objetivo.
  • Nivel 1: Realizado (Performed): El proceso se realiza y alcanza su objetivo.
  • Nivel 2: Gestionado (Managed): El proceso se realiza, se planifica, monitoriza y ajusta; los productos de trabajo se establecen y controlan.
  • Nivel 3: Establecido (Established): El proceso se realiza utilizando un proceso definido, adaptado de un conjunto de procesos estándar de la organización; se recopilan datos para análisis y mejora.
  • Nivel 4: Predecible (Predictable): El proceso opera dentro de límites predecibles para alcanzar los objetivos del proceso; se utilizan mediciones cuantitativas.
  • Nivel 5: Optimizado (Optimizing): Se mejora continuamente el desempeño del proceso basándose en la comprensión cuantitativa de las variaciones y la causalidad.

Estándares IEEE

El Instituto de Ingenieros Eléctricos y Electrónicos (IEEE) publica una amplia gama de estándares para ingeniería de software. A diferencia de ISO 25000 que define un modelo de calidad del producto o ISO 15504 que define un marco para la evaluación de procesos, los estándares IEEE a menudo proporcionan guías detalladas y formatos para artefactos específicos del ciclo de vida del software.

Ejemplos comunes incluyen:

  • IEEE 829: Estándar para la documentación de pruebas de software (planes de prueba, casos de prueba, informes, etc.).
  • IEEE 1008: Estándar para pruebas unitarias.
  • IEEE 1016: Estándar para descripción de diseño de software.
  • IEEE 1028: Estándar para revisiones y auditorías de software.
  • IEEE 1062: Estándar para la mejora de la calidad del software.

Son muy prácticos y específicos, a menudo utilizados como referencia para definir procedimientos y plantillas dentro de una organización.

Para abordar este objetivo, ¿qué combinación de estándares es la más relevante para iniciar? (Selecciona dos opciones, una de cada tipo principal de estándar).




Conclusión

ISO/IEC 25000 nos ayuda a entender y definir qué significa la calidad del producto software. ISO/IEC 15504 nos permite evaluar y mejorar los procesos con los que creamos software. Los estándares IEEE proporcionan guías prácticas y formatos para actividades y artefactos específicos dentro de esos procesos. Utilizar estos estándares de manera conjunta es fundamental para construir y mantener software de alta calidad.
